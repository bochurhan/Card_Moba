## 一、核心设计原则（强制遵守）
1.  **单一职责原则**：每个模块仅负责单一核心能力，禁止跨模块编写业务逻辑
2.  **全复用原则**：卡牌、Buff、触发器的效果逻辑100%复用同一套效果原子体系，禁止重复开发同类逻辑
3.  **数据驱动原则**：所有玩法内容可通过配置表实现，无需修改核心代码
4.  **时序锁死原则**：所有结算流程、触发逻辑有明确的优先级与时序规则，禁止越级、乱序执行
5.  **可追溯原则**：所有战斗操作、数值变化全链路可记录、可复盘、可回滚，数值分段全透明
6.  **读写隔离原则**：战斗上下文仅核心结算引擎、效果Handler可执行写操作，其余模块仅拥有只读权限，杜绝脏数据
7.  **全量行为可控原则**：所有核心玩家行为均有统一拦截校验节点，支持配置化实现行为限制，无硬编码黑盒逻辑
8.  **全链路数值可修正原则**：所有效果数值支持配置化的实时修正，无需修改Handler核心代码，适配属性加成、数值增减等通用需求

## 二、核心术语定义（新增/补全版）
| 术语 | 标准化定义 |
|------|------------|
| 瞬策牌 | 打出后立即进入结算流程，无延时等待的卡牌，结算完成前仅开放专属反制窗口 |
| 定策牌 | 打出后进入定策区锁定，等待指定结算窗口统一批量结算的卡牌，结算层级绑定到效果原子 |
| 状态牌 | 特殊卡牌，无法主动打出，仅可通过效果生成，绑定触发型效果，生命周期受区域规则管控 |
| 效果原子 | 不可拆分的最小效果执行单元，一个效果原子对应唯一的执行Handler，是所有战斗效果的基础 |
| 战斗上下文 | 单场战斗的唯一全局数据入口，存储战斗全量状态、管理器实例、临时数据与日志 |
| 触发器 | 「事件-条件-动作」的统一执行载体，负责管理所有延时、触发、响应式、行为拦截逻辑 |
| 事件总线 | 战斗内所有事件的统一派发、订阅中心，是模块间解耦的核心，也是日志、统计的数据源 |
| 动态参数映射 | 效果原子支持从上下文、前置效果执行结果、事件参数中读取动态数值，替代静态配置参数，支持四则运算表达式与嵌套对象属性读取 |
| 核心行为前置拦截 | 所有玩家核心操作执行前，必须经过统一的权限校验节点，支持配置化拦截/限制行为 |
| 分路 | 战场的横向区域划分，用于限定卡牌目标范围、单位生效区域 |
| 数值修正器 | 配置化的数值修正单元，可实时修改效果原子的入参数值，适配属性加成、数值增减等通用需求 |
| 消耗区 | 卡牌特殊存放区域，卡牌使用后不进弃牌堆、UI不展示、永久记录，支持与其他卡牌效果互动 |
| DOT伤害 | 持续伤害，每回合固定时机触发，与直伤、反弹伤害、真实伤害做类型区分，支持独立的免疫/减免规则 |
| 手动选择目标 | 卡牌释放前需玩家手动选择目标，支持最小/最大数量强制校验，不满足则无法释放 |
| 卡牌生成 | 通过效果动态创建指定卡牌实例，并放入目标卡牌区域的核心能力 |
| 卡牌复刻 | 无消耗重复执行已打出卡牌的完整结算流程，支持配置是否继承原目标、是否触发打出事件 |

## 三、整体架构总览
采用分层架构，从上到下依赖关系单向传递，禁止下层反向依赖上层
```
┌─────────────────────────────────────────────────────┐
│ 接入层：玩家操作输入、前端表现、战斗回放、手动选目标交互 │
├─────────────────────────────────────────────────────┤
│ 调度层：战斗结算引擎（含核心行为拦截）、事件总线    │
├─────────────────────────────────────────────────────┤
│ 业务层：卡牌模块、Buff模块、触发器模块、统计模块    │
├─────────────────────────────────────────────────────┤
│ 基础能力层：战斗上下文、目标解析器、条件判断器、Handler池、动态参数解析器、数值修正器管理器 │
├─────────────────────────────────────────────────────┤
│ 配置层：策划配置表、常量配置、版本管理              │
├─────────────────────────────────────────────────────┤
│ 日志层：结构化战斗日志、异常日志、调试工具          │
└─────────────────────────────────────────────────────┘
```

## 四、核心模块详细设计（调整后全量版）
### 4.1 战斗上下文 BattleContext
#### 核心定位
单场战斗的**唯一全局数据入口**，全局单例，仅战斗结算引擎、效果Handler可执行写操作，其余模块仅可读。
#### 标准化属性定义（C#伪代码）
```csharp
/// <summary>
/// 单场战斗全局上下文
/// </summary>
public class BattleContext
{
    #region 战斗基础信息
    public string BattleId { get; set; } // 战斗全局唯一ID
    public int CurrentRound { get; set; } // 当前回合数

    public BattleStatus BattleStatus { get; set; } // 战斗状态
    public string FirstPlayerId { get; set; } // 先手玩家ID
    public string CurrentRoundPlayerId { get; set; } // 当前回合归属玩家ID
    public int MaxLaneCount { get; set; } = 2; // 战场总分路数
    #endregion
    #region 玩家实体数据
    /// <summary>
    /// 战斗内所有玩家数据，Key=玩家ID
    /// </summary>
    public Dictionary<string, PlayerData> Players { get; set; } = new();
    #endregion
    #region 全局管理器挂载
    public IEventBus EventBus { get; set; } // 事件总线
    public ITriggerManager TriggerManager { get; set; } // 触发器管理器
    public IStatManager StatManager { get; set; } // 统计值管理器
    public ITargetResolver TargetResolver { get; set; } // 目标解析器
    public IConditionChecker ConditionChecker { get; set; } // 条件判断器
    public IEffectHandlerPool HandlerPool { get; set; } // 效果Handler池
    public IDynamicParamResolver DynamicParamResolver { get; set; } // 动态参数解析器
    public IValueModifierManager ValueModifierManager { get; set; } // 数值修正器管理器
    #endregion
    #region 全局数值修正器
    /// <summary>
    /// 全局生效的数值修正器
    /// </summary>
    public List<ValueModifier> GlobalValueModifiers { get; set; } = new();
    #endregion
    #region 临时结算数据
    /// <summary>
    /// 结算阶段临时数据，阶段结束后自动清空
    /// </summary>
    public Dictionary<string, object> TempSettleData { get; set; } = new();

    #region 不可篡改战斗日志
    /// <summary>
    /// 全量战斗日志，仅追加不可修改
```

#### 核心规则
1.  每回合结束、每个结算阶段完成后，自动生成战斗数据快照，用于异常回滚、问题排查与战斗复盘
2.  临时结算数据仅在当前结算阶段有效，阶段结束后必须强制清空，禁止跨阶段携带数据
3.  所有战斗内的实体数据修改，必须通过上下文执行，禁止直接修改实体对象属性
4.  玩家分路ID、回合归属玩家ID仅可在战斗初始化/回合切换时设置，战斗过程中禁止修改，仅可通过效果修改目标筛选的分路范围
5.  消耗区卡牌仅可通过配置的效果移动，禁止手动修改，进入消耗区的卡牌永久记录，战斗结束前不可销毁
6.  回合切换时，自动清空玩家本回合卡牌打出记录，禁止跨回合携带

### 4.2 事件总线 EventBus
#### 核心定位
战斗内所有事件的统一派发、订阅、注销中心，是模块解耦的核心，也是战斗日志、统计值更新、触发器触发的唯一数据源。
#### 标准化接口定义（新增/补全版）
```csharp
/// <summary>
/// 战斗事件枚举（强制标准化，禁止自定义非枚举事件）
/// </summary>
public enum BattleEventType
{
    // 战斗流程事件
    BattleStart, RoundStart, RoundEnd, PhaseChange, BattleEnd,
    // 卡牌事件（新增CardGenerate、CardReplay、StateCardTrigger）
    CardDraw, CardPlay, CardCountered, CardSettleComplete, CardDiscard, CardMove, CardConsumed, CardGenerate, CardReplay, StateCardTrigger,
    // 效果事件
    EffectBeforeExecute, EffectAfterExecute, EffectCountered, EffectInvalidated,
    // 数值事件
    DamageBeforeDeal, DamageAfterDeal, HpChange, ShieldChange, CostChange, AttributeChange,
    // Buff事件
    BuffAdd, BuffRemove, BuffLayerChange, BuffTrigger,
    // 玩家操作事件
    PlayerOperation, PlayerSurrender, PlayerTargetSelectComplete,
    // 核心行为拦截事件
    ActionBeforeExecute, ActionAfterExecute
}
/// <summary>
/// 战斗核心行为枚举（所有可被拦截的玩家操作，新增ReplayCard）
/// </summary>
public enum BattleCoreAction
{
    DrawCard, PlayCard, DealDamage, CastSkill, GainCost, Heal, PlayInstantCard, PlayStrategyCard, DiscardCard, ReplayCard
}
/// <summary>
/// 事件总线接口
/// </summary>
public interface IEventBus
{
    /// <summary>
    /// 订阅事件
    /// </summary>
    void Subscribe(BattleEventType eventType, Action<BattleEvent> callback);
    
    /// <summary>
    /// 注销订阅
    /// </summary>
    void Unsubscribe(BattleEventType eventType, Action<BattleEvent> callback);
    
    /// <summary>
    /// 派发事件（先写入日志，再执行回调）
    /// </summary>
    void Dispatch(BattleEvent eventData);
}
/// <summary>
/// 战斗事件数据体
/// </summary>
public class BattleEvent
{
    public BattleEventType EventType { get; set; }
    public int TriggerRound { get; set; }
    public BattlePhase TriggerPhase { get; set; }
    public string SourceId { get; set; } // 事件来源ID
    public List<string> TargetIds { get; set; } // 事件目标ID
    public BattleCoreAction? ActionType { get; set; } // 核心行为类型（拦截事件必填）
    public Dictionary<string, object> EventArgs { get; set; } = new();
}
```
#### 核心规则
1.  事件派发遵循「先记录，后执行」原则：事件先写入战斗日志永久存储，再执行订阅回调
2.  禁止在事件回调中修改事件源数据，仅可读取，杜绝脏数据
3.  同一事件的回调执行顺序，由触发器/订阅者的优先级决定，优先级越高越先执行
4.  所有核心行为执行前，必须派发`ActionBeforeExecute`事件，执行完成后派发`ActionAfterExecute`事件
5.  卡牌进入消耗区时，必须派发`CardConsumed`事件，触发相关统计与互动效果
6.  卡牌生成、复刻、状态牌触发时，必须派发对应事件，同步更新统计与日志

### 4.3 卡牌模块
#### 4.3.1 卡牌类型与结算规则
| 卡牌类型 | 核心定义 | 结算规则 | 反制窗口 |
|----------|----------|----------|----------|
| 瞬策牌   | 即时结算卡牌，打出后立即执行结算 | 单卡牌原子化结算，结算完成前不响应其他卡牌操作（连锁反制除外） | 打出后、结算前开放专属反制窗口，仅可响应反制类效果 |
| 定策牌   | 延时结算卡牌，打出后进入定策区锁定 | 严格遵循「反制→防御→伤害→其他」4个层级批量结算，层级绑定到效果原子，不可越级 | 仅在定策结算阶段的「反制层」可被反制，反制成功后指定效果/整牌失效 |
| 状态牌   | 特殊限制卡牌，仅可通过效果生成，无法主动打出 | 仅触发绑定的触发器效果，不进入主动结算流程 | 无主动反制窗口，仅可通过卡牌移除效果销毁 |
#### 4.3.2 定策结算层级强制规则
**核心规则：结算层级从卡牌维度改为效果原子维度**，单张定策牌的多个效果原子可分属不同层级，彻底解决时序混乱问题
1.  层级优先级锁死：**反制层 > 防御层 > 伤害层 > 其他层**，同层级内不可跨层级触发效果
2.  全局结算排序规则：先按「层级优先级」排序 → 同层级按「效果原子配置优先级」排序 → 同优先级按「卡牌打出顺序」正序排序（先打出先执行）
3.  层级边界与允许效果类型定义：
    | 层级 | 允许执行的效果类型 | 核心规则 |
    |------|--------------------|----------|
    | 反制层 | 卡牌反制、效果无效化、结算中断 | 被反制的卡牌/效果直接标记失效，移出结算队列，不进入后续层级 |
    | 防御层 | 护盾添加、伤害免疫/减免、生命恢复、防御类属性修改 | 确保伤害结算前，所有防御属性已生效 |
    | 伤害层 | 直伤、持续伤害、伤害反弹、伤害类属性修改 | 统一在防御属性生效后计算最终伤害，伤害数值分段全记录 |
    | 其他层 | 资源修改层（费用/抽牌/弃牌）→ Buff管理层 → 特殊规则层（回合跳过等） | 拆分为3个子层级，按固定顺序执行，禁止乱序 |

### 4.4 效果原子与Handler体系（全量补全版）
#### 核心定位
所有战斗效果的最小执行单元，实现「配置化组合效果」的核心，卡牌、Buff、触发器的效果100%复用该体系。

```csharp
/// <summary>
/// 效果Handler基接口
/// </summary>
public interface IEffectHandler
{
    EffectType HandleType { get; }
    /// <summary>
    /// 执行效果
    /// </summary>
    EffectResult Execute(BattleContext context, EffectUnit effect, Entity source, List<Entity> targets);
}
/// <summary>
/// 通用属性修改Handler
/// </summary>
public class AttributeModifyHandler : IEffectHandler
{
    public EffectType HandleType => EffectType.AttributeModify;
    public EffectResult Execute(BattleContext context, EffectUnit effect, Entity source, List<Entity> targets)
    {
        var result = new EffectResult();
        result.BeforeSnapshot = new Dictionary<string, ValueSnapshot>();
        // 读取参数
        var attrName = effect.EffectParams["AttributeName"].ToString();
        var operateType = (AttributeOperateType)effect.EffectParams["OperateType"];
        var value = Convert.ToDecimal(effect.EffectParams["Value"]);
        try
        {
            foreach (var target in targets)
            {
                if (target is not PlayerData player) continue;
                // 记录执行前快照
                var originalValue = GetAttributeValue(player, attrName);
                result.BeforeSnapshot.Add($"{player.PlayerId}_{attrName}", new ValueSnapshot
                {
                    TargetId = player.PlayerId,
                    ValueKey = attrName,
                    OriginalValue = originalValue
                });
                // 执行属性修改
                var newValue = CalculateNewValue(originalValue, operateType, value);
                SetAttributeValue(player, attrName, newValue);
                result.AfterValue.Add($"{player.PlayerId}_{attrName}", newValue);
            }
            result.IsSuccess = true;
        }
        catch (Exception ex)
        {
            result.IsSuccess = false;
            result.ErrorMsg = ex.Message;
        }
        return result;
    }
    // 辅助方法：获取属性值、计算新值、设置属性值
    private decimal GetAttributeValue(PlayerData player, string attrName)
    {
        return attrName switch
        {
            "AttackPower" => player.AttackPower,
            "Agility" => player.Agility,
            "MaxHp" => player.MaxHp,
            "CurrentHp" => player.CurrentHp,
            "CurrentShield" => player.CurrentShield,
            _ => 0
        };
    }
    private decimal CalculateNewValue(decimal original, AttributeOperateType operateType, decimal value)
    {
        return operateType switch
        {
            AttributeOperateType.Add => original + value,
            AttributeOperateType.Subtract => original - value,
            AttributeOperateType.Multiply => original * value,
            AttributeOperateType.Divide => value == 0 ? original : original / value,
            AttributeOperateType.Set => value,
            AttributeOperateType.PercentAdd => original * (1 + value / 100),
            AttributeOperateType.PercentSubtract => original * (1 - value / 100),
            _ => original
        };
    }
    private void SetAttributeValue(PlayerData player, string attrName, decimal newValue)
    {
        switch (attrName)
        {
            case "AttackPower":
                player.AttackPower = (int)Math.Round(newValue);
                break;
            case "Agility":
                player.Agility = (int)Math.Round(newValue);
                break;
            case "MaxHp":
                player.MaxHp = (int)Math.Round(newValue);
                break;
            case "CurrentHp":
                player.CurrentHp = (int)Math.Round(newValue);
                break;
            case "CurrentShield":
                player.CurrentShield = (int)Math.Round(newValue);
                break;
        }
    }
}
/// <summary>
/// 直伤效果Handler（标准化伤害结算流程，支持多伤害类型）
/// </summary>
public class DirectDamageHandler : IEffectHandler
{
    public EffectType HandleType => EffectType.DirectDamage;
    public EffectResult Execute(BattleContext context, EffectUnit effect, Entity source, List<Entity> targets)
    {
        var result = new EffectResult();
        result.BeforeSnapshot = new Dictionary<string, ValueSnapshot>();
        var damageDetail = new DamageDetail();
        // 1. 读取基础伤害参数
        var baseDamage = Convert.ToInt32(effect.EffectParams["BaseDamage"]);
        var damageType = (DamageType)effect.EffectParams["DamageType"];
        var sourcePlayer = source as PlayerData;
        try
        {
            foreach (var target in targets)
            {
                if (target is not PlayerData targetPlayer) continue;
                // 2. 记录执行前快照
                result.BeforeSnapshot.Add($"{targetPlayer.PlayerId}_Hp", new ValueSnapshot
                {
                    TargetId = targetPlayer.PlayerId,
                    ValueKey = "CurrentHp",
                    OriginalValue = targetPlayer.CurrentHp
                });
                result.BeforeSnapshot.Add($"{targetPlayer.PlayerId}_Shield", new ValueSnapshot
                {
                    TargetId = targetPlayer.PlayerId,
                    ValueKey = "CurrentShield",
                    OriginalValue = targetPlayer.CurrentShield
                });
                // 3. 标准化伤害计算公式
                var finalDamage = CalculateFinalDamage(baseDamage, damageType, sourcePlayer, targetPlayer);
                var blockDamage = damageType == DamageType.True ? 0 : Math.Min(finalDamage, targetPlayer.CurrentShield);
                var realDamage = finalDamage - blockDamage;
                var overflowDamage = Math.Max(realDamage - targetPlayer.CurrentHp, 0);
                // 4. 写入伤害明细，用于动态参数读取与日志
                damageDetail.TotalBaseDamage += baseDamage;
                damageDetail.TotalFinalDamage += finalDamage;
                damageDetail.TotalBlockDamage += blockDamage;
                damageDetail.TotalRealDamage += realDamage;
                damageDetail.TotalOverflowDamage += overflowDamage;
                damageDetail.DamageType = damageType;
                // 5. 执行数值修改
                if (damageType != DamageType.True)
                {
                    targetPlayer.CurrentShield = Math.Max(0, targetPlayer.CurrentShield - blockDamage);
                }
                targetPlayer.CurrentHp = Math.Max(0, targetPlayer.CurrentHp - realDamage);
                // 6. 记录执行后数值
                result.AfterValue.Add($"{targetPlayer.PlayerId}_Hp", targetPlayer.CurrentHp);
                result.AfterValue.Add($"{targetPlayer.PlayerId}_Shield", targetPlayer.CurrentShield);
            }
            // 7. 伤害明细存入结果，用于动态参数映射
            result.AfterValue.Add("DamageDetail", damageDetail);
            result.IsSuccess = true;
        }
        catch (Exception ex)
        {
            result.IsSuccess = false;
            result.ErrorMsg = ex.Message;
        }
        return result;
    }
    // 辅助方法：计算最终伤害
    private int CalculateFinalDamage(int baseDamage, DamageType damageType, PlayerData source, PlayerData target)
    {
        // 基础伤害+力量加成
        var damage = baseDamage + source.AttackPower;
        // 伤害减免/免疫判断（按伤害类型区分）
        // 从Buff、上下文读取对应伤害类型的减免倍率，默认1.0
        return Math.Max(0, (int)Math.Round(damage));
    }
}
/// <summary>
/// DOT持续伤害Handler（独立实现，与直伤规则隔离）
/// </summary>
public class DOTDamageHandler : IEffectHandler
{
    public EffectType HandleType => EffectType.DOTDamage;
    public EffectResult Execute(BattleContext context, EffectUnit effect, Entity source, List<Entity> targets)
    {
        var result = new EffectResult();
        result.BeforeSnapshot = new Dictionary<string, ValueSnapshot>();
        var damageDetail = new DamageDetail();
        // 读取参数
        var baseDamage = Convert.ToInt32(effect.EffectParams["BaseDamage"]);
        var damageType = DamageType.DOT;
        var sourcePlayer = source as PlayerData;
        try
        {
            foreach (var target in targets)
            {
                if (target is not PlayerData targetPlayer) continue;
                // 快照记录
                result.BeforeSnapshot.Add($"{targetPlayer.PlayerId}_Hp", new ValueSnapshot
                {
                    TargetId = targetPlayer.PlayerId,
                    ValueKey = "CurrentHp",
                    OriginalValue = targetPlayer.CurrentHp
                });
                // DOT伤害无视护盾，仅受DOT专属减免/免疫影响
                var finalDamage = Math.Max(0, baseDamage);
                var realDamage = Math.Min(finalDamage, targetPlayer.CurrentHp);
                var overflowDamage = finalDamage - realDamage;
                // 写入伤害明细
                damageDetail.TotalBaseDamage += baseDamage;
                damageDetail.TotalFinalDamage += finalDamage;
                damageDetail.TotalRealDamage += realDamage;
                damageDetail.TotalOverflowDamage += overflowDamage;
                damageDetail.DamageType = damageType;
                // 执行伤害
                targetPlayer.CurrentHp = Math.Max(0, targetPlayer.CurrentHp - realDamage);
                result.AfterValue.Add($"{targetPlayer.PlayerId}_Hp", targetPlayer.CurrentHp);
            }
            result.AfterValue.Add("DamageDetail", damageDetail);
            result.IsSuccess = true;
        }
        catch (Exception ex)
        {
            result.IsSuccess = false;
            result.ErrorMsg = ex.Message;
        }
        return result;
    }
}

            result.IsSuccess = true;

            result.IsSuccess = false;
            result.ErrorMsg = ex.Message;
        }
        return result;
    }

/// <summary>
/// 卡牌生成Handler（新增，适配超频等效果）
/// </summary>
public class CardGenerateHandler : IEffectHandler
{
    public EffectType HandleType => EffectType.CardGenerate;
    public EffectResult Execute(BattleContext context, EffectUnit effect, Entity source, List<Entity> targets)
    {
        var result = new EffectResult();
        result.BeforeSnapshot = new Dictionary<string, ValueSnapshot>();
        // 读取参数
        var targetCardId = effect.EffectParams["TargetCardId"].ToString();
        var generateCount = Convert.ToInt32(effect.EffectParams.GetValueOrDefault("GenerateCount", 1));
        var targetArea = (CardArea)effect.EffectParams["TargetArea"];
        var sourcePlayer = source as PlayerData;
        try
        {
            foreach (var target in targets)
            {
                if (target is not PlayerData targetPlayer) continue;
                // 生成卡牌实例
                for (int i = 0; i < generateCount; i++)
                {
                    var newCard = CardConfigManager.GetCardConfig(targetCardId).Clone();
                    newCard.OwnerId = targetPlayer.PlayerId;
                    newCard.CurrentArea = targetArea;
                    // 加入目标区域
                    AddCardToArea(targetPlayer, newCard, targetArea);
                    // 注册卡牌绑定的触发器
                    foreach (var trigger in newCard.BindTriggers)
                    {
                        context.TriggerManager.Register(context, trigger);
                    }
                    // 派发事件
                    context.EventBus.Dispatch(new BattleEvent
                    {
                        EventType = BattleEventType.CardGenerate,
                        TriggerRound = context.CurrentRound,
                        TriggerPhase = context.CurrentPhase,
                        SourceId = source.Id,
                        TargetIds = new List<string> { targetPlayer.PlayerId },
                        EventArgs = new Dictionary<string, object> { { "CardId", targetCardId }, { "TargetArea", targetArea } }
                    });
                }
                result.AfterValue.Add($"{targetPlayer.PlayerId}_GenerateCount", generateCount);
            }
            result.IsSuccess = true;
        }
        catch (Exception ex)
        {
            result.IsSuccess = false;
            result.ErrorMsg = ex.Message;
        }
        return result;
    }
    // 辅助方法：将卡牌加入目标区域
    private void AddCardToArea(PlayerData player, CardData card, CardArea area)
    {
        switch (area)
        {
            case CardArea.Deck:
                player.Deck.Add(card);
                break;
            case CardArea.Hand:
                player.Hand.Add(card);
                break;
            case CardArea.Cemetery:
                player.Cemetery.Add(card);
                break;
            case CardArea.Consumed:
                player.ConsumedCards.Add(card);
                break;
            case CardArea.Removed:
                player.RemovedCards.Add(card);
                break;
        }
    }
}
/// <summary>
/// 卡牌复刻Handler（新增，适配回响型态效果）
/// </summary>
public class CardReplayHandler : IEffectHandler
{
    public EffectType HandleType => EffectType.CardReplay;
    public EffectResult Execute(BattleContext context, EffectUnit effect, Entity source, List<Entity> targets)
    {
        var result = new EffectResult();
        // 读取参数
        var sourceCardId = effect.EffectParams["SourceCardId"].ToString();
        var skipCost = Convert.ToBoolean(effect.EffectParams.GetValueOrDefault("SkipCost", true));
        var inheritTarget = Convert.ToBoolean(effect.EffectParams.GetValueOrDefault("InheritTarget", true));
        var triggerPlayEvent = Convert.ToBoolean(effect.EffectParams.GetValueOrDefault("TriggerPlayEvent", false));
        var sourcePlayer = source as PlayerData;
        try
        {
            // 获取源卡牌配置
            var sourceCard = CardConfigManager.GetCardConfig(sourceCardId).Clone();
            sourceCard.OwnerId = sourcePlayer.PlayerId;
            // 执行前置校验
            var engine = new BattleSettleEngine();
            if (!engine.PreActionCheck(context, BattleCoreAction.ReplayCard, source))
            {
                result.IsSuccess = false;
                result.ErrorMsg = "复刻行为被拦截";
                return result;
            }
            // 执行复刻结算
            if (sourceCard.CardType == CardType.Instant)
            {
                engine.ExecuteInstantCardSettle(context, sourceCard, sourcePlayer, skipCost, inheritTarget, triggerPlayEvent);
            }
            else if (sourceCard.CardType == CardType.Strategy)
            {
                engine.ExecuteStrategyCardReplaySettle(context, sourceCard, sourcePlayer, skipCost, inheritTarget);
            }
            // 派发事件
            context.EventBus.Dispatch(new BattleEvent
            {
                EventType = BattleEventType.CardReplay,
                TriggerRound = context.CurrentRound,
                TriggerPhase = context.CurrentPhase,
                SourceId = source.Id,
                TargetIds = new List<string> { sourcePlayer.PlayerId },
                EventArgs = new Dictionary<string, object> { { "CardId", sourceCardId }, { "SkipCost", skipCost } }
            });
            result.IsSuccess = true;
        }
        catch (Exception ex)
        {
            result.IsSuccess = false;
            result.ErrorMsg = ex.Message;
        }
        return result;
    }
}
```

#### 4.4.4 执行时序强制规范（更新版）
每个Handler必须严格遵循以下执行顺序，禁止越级、跳步：
1.  **动态参数解析**：调用动态参数解析器，根据`DynamicParamMap`读取上下文/前置效果数值、解析表达式，覆盖静态参数
2.  **数值修正**：调用数值修正器管理器，执行所有匹配的数值修正，得到最终入参
3.  **前置条件校验**：调用条件判断器检查执行条件，不满足则直接跳过，记录日志
4.  **目标解析**：调用目标解析器获取合法目标，无合法目标则按配置规则跳过/终止
5.  **事件派发**：派发`EffectBeforeExecute`事件，触发反制、伤害减免等响应逻辑
6.  **合法性校验**：检查是否被反制/无效化，已失效则直接终止，记录日志
7.  **快照留存**：记录目标执行前的数值快照，用于回滚与日志追溯
8.  **循环执行核心逻辑**：按`ExecuteCount`循环执行，每次循环根据`ReResolveTargetPerExecute`决定是否重解析目标
9.  **结果记录**：生成执行结果，写入战斗日志与临时结算数据
10. **事件派发**：派发`EffectAfterExecute`事件，触发吸血、统计更新等后置逻辑
#### 4.4.5 回滚规则
1.  单效果原子：执行失败时，可通过`CanRollback`配置是否回滚到执行前快照
2.  整卡牌结算：反制成功时，未执行的效果全部终止，已执行完成的效果不回滚
3.  全局异常：战斗出现致命异常时，回滚到上一个回合结束的完整快照
4.  循环执行异常：循环执行中途失败时，已执行的循环步骤不回滚，终止后续循环

### 4.5 目标解析器 TargetResolver
#### 核心定位
统一处理所有效果的目标筛选、合法性校验、动态解析，支持策划配置化实现任意复杂的目标选择逻辑，含手动选目标能力。
#### 标准化定义
```csharp
/// <summary>
/// 目标实体类型枚举
/// </summary>
```

### 4.7 触发器系统（完善版）
#### 核心定位
「事件-条件-动作」的统一执行载体，负责管理所有延时、触发、响应式、行为拦截逻辑，是Buff、被动效果、反制效果的核心载体。
#### 4.7.1 生命周期管理方案
触发器分为四类，生命周期完全分开，由全局`TriggerManager`统一管控，禁止任何模块自行注册/注销触发器。
| 触发器类型 | 生命周期规则 | 适用场景 |
|------------|--------------|----------|
| 绑定型触发器 | 生命周期与宿主完全绑定，宿主销毁/失效/区域变化时，自动注销/暂停所有回调 | Buff自带的触发效果、卡牌被动效果、单位光环效果、行为拦截效果、状态牌绑定效果 |
| 独立型触发器 | 独立生命周期，触发完成/超出时间窗口后自动注销 | 定策牌延时结算、一次性触发效果、全局统计触发器 |
| 全局战斗触发器 | 宿主类型为Battle，生命周期与战斗完全绑定，战斗结束才注销 | 毒雾等全场永久生效的触发效果 |
| 卡牌宿主触发器 | 生命周期与卡牌绑定，卡牌区域变化时自动激活/暂停，卡牌销毁时自动注销 | 灼伤状态牌的回合结束触发效果、卡牌在手牌区的被动效果 |
#### 4.7.2 生命周期状态机（强制流转，不可越级）
`注册 → 待激活 → 激活中 → 已触发/已过期 → 已注销`
1.  注册：仅可通过`TriggerManager.Register()`注册，必须明确生命周期类型、触发事件、优先级
2.  待激活：已注册，等待对应事件派发
3.  激活中：事件已派发，正在执行条件判断与回调
4.  已触发：一次性触发器触发完成，或次数型触发器达到次数上限，进入待注销队列
5.  已过期：超出时间窗口的触发器，回合结束自动进入待注销队列
6.  已注销：`TriggerManager`每帧清理待注销队列，彻底移除触发器，释放内存

5.  行为拦截规则：拦截触发器返回false时，直接终止对应核心行为的执行，记录拦截日志
6.  全局战斗触发器仅可通过`ClearAllBattleTriggers`在战斗结束时清理，中途不会因卡牌销毁、区域移动而失效

```csharp
/// <summary>
/// 数值修正器管理器实现
/// </summary>
public class ValueModifierManager : IValueModifierManager
{
    // 全局自动修正规则字典
    private Dictionary<string, Func<BattleContext, Entity, ValueModifier>> _globalAutoModifiers = new();
    public Dictionary<string, object> ModifyEffectParams(BattleContext context, EffectUnit effect, Entity source)
    {
        // 1. 复制原始参数
        var finalParams = new Dictionary<string, object>(effect.EffectParams);
        // 2. 获取所有匹配当前效果的修正器（全局+自动修正+来源玩家Buff绑定）
        var matchModifiers = GetMatchModifiers(context, effect, source);
        // 3. 按优先级排序
        matchModifiers = matchModifiers.OrderByDescending(m => m.Priority).ToList();
        // 4. 按修正类型优先级执行：加法 > 百分比 > 乘法 > 赋值
        var addModifiers = matchModifiers.Where(m => m.ModifyType == ValueModifyType.Add).ToList();
        var percentModifiers = matchModifiers.Where(m => m.ModifyType == ValueModifyType.PercentAdd).ToList();
        var multiplyModifiers = matchModifiers.Where(m => m.ModifyType == ValueModifyType.Multiply).ToList();
        var setModifiers = matchModifiers.Where(m => m.ModifyType == ValueModifyType.Set).ToList();
        // 5. 逐个执行修正
        foreach (var modifier in addModifiers.Concat(percentModifiers).Concat(multiplyModifiers).Concat(setModifiers))
        {
            // 校验生效条件
            if (modifier.ValidCondition != null && !context.ConditionChecker.Check(context, modifier.ValidCondition, source))
                continue;
            // 执行参数修正
            if (!finalParams.ContainsKey(modifier.TargetParamKey)) continue;
            var originalValue = Convert.ToDecimal(finalParams[modifier.TargetParamKey]);
            var newValue = modifier.ModifyType switch
            {
                ValueModifyType.Add => originalValue + modifier.ModifyValue,
                ValueModifyType.PercentAdd => originalValue * (1 + modifier.ModifyValue / 100),
                ValueModifyType.Multiply => originalValue * modifier.ModifyValue,
                ValueModifyType.Set => modifier.ModifyValue,
                _ => originalValue
            };
            finalParams[modifier.TargetParamKey] = newValue;
        }
        return finalParams;
    }
    // 辅助方法：获取匹配的修正器
    private List<ValueModifier> GetMatchModifiers(BattleContext context, EffectUnit effect, Entity source)
    {
        var modifiers = new List<ValueModifier>();
        // 全局修正器
        modifiers.AddRange(context.GlobalValueModifiers.Where(m => m.TargetEffectType == effect.EffectType));
        // 全局自动修正器
        foreach (var autoModifier in _globalAutoModifiers.Values)
        {
            var modifier = autoModifier.Invoke(context, source);
            if (modifier.TargetEffectType == effect.EffectType)
            {
                modifiers.Add(modifier);
            }
        }
        // 来源玩家的Buff绑定修正器
        if (source is PlayerData player)
        {
            foreach (var buff in player.ActiveBuffs)
            {
                modifiers.AddRange(buff.ValueModifiers.Where(m => m.TargetEffectType == effect.EffectType));
            }
        }
        return modifiers;
    }
    // 其余接口实现略
}
```
#### 核心修正优先级规则
**加法修正 > 百分比加成 > 乘法修正 > 强制赋值**，同类型修正按优先级从高到低执行，避免数值计算混乱。
#### 全局自动修正规则（新增）
战斗初始化时，自动注册「敏捷属性对护盾值的加成」全局修正器，实现1点敏捷=卡牌护盾值+1的自动适配，无需单卡牌配置。

```csharp
/// <summary>
/// 动态参数解析器实现
/// </summary>
public class DynamicParamResolver : IDynamicParamResolver
{
    public void ResolveAndOverride(BattleContext context, EffectUnit currentEffect, List<EffectResult> preEffectResults, Entity source)
    {
        if (currentEffect.DynamicParamMap == null || currentEffect.DynamicParamMap.Count == 0)
            return;
        foreach (var paramMap in currentEffect.DynamicParamMap)
        {
            var paramName = paramMap.Key;
            var pathOrExpression = paramMap.Value;
            object value;
            // 区分表达式与路径
            if (pathOrExpression.StartsWith("{{") && pathOrExpression.EndsWith("}}"))
            {
                // 解析四则运算表达式
                var expression = pathOrExpression.Trim('{', '}');
                value = ResolveExpression(expression, context, source, preEffectResults);
            }
            else
            {
                // 解析路径参数，支持嵌套对象二级属性读取
                var pathParts = pathOrExpression.Split('.');
                value = pathParts[0] switch
                {
                    "preEffect" => GetPreEffectValue(pathParts, preEffectResults),
                    "context" => GetContextValue(pathParts, context, source),
                    "event" => GetEventValue(pathParts, context),
                    _ => null
                };
            }
            if (value != null)
            {
                currentEffect.EffectParams[paramName] = value;
            }
        }
    }
    // 辅助方法：从前置效果结果中读取数值，支持嵌套对象属性
    private object GetPreEffectValue(string[] pathParts, List<EffectResult> preEffectResults)
    {
        if (pathParts.Length < 3) return null;
        var effectId = pathParts[1];
        var fieldName = pathParts[2];
        var targetResult = preEffectResults.FirstOrDefault(r => r.EffectId == effectId);
        if (targetResult == null) return null;
        // 支持嵌套属性读取，如preEffect.damage_01.DamageDetail.TotalRealDamage
        if (pathParts.Length == 3)
        {
            return targetResult.AfterValue.TryGetValue(fieldName, out var value) ? value : null;
        }
        else if (pathParts.Length == 4 && targetResult.AfterValue.TryGetValue(fieldName, out var objValue))
        {
            var subFieldName = pathParts[3];
            return objValue.GetType().GetProperty(subFieldName)?.GetValue(objValue);
        }
        return null;
    }
    // 辅助方法：从上下文中读取数值
    private object GetContextValue(string[] pathParts, BattleContext context, Entity source)
    {
        if (pathParts.Length < 3) return null;
        return pathParts[1] switch
        {
            "stat" => GetStatValue(pathParts, context, source),
            "self" => GetSelfAttributeValue(pathParts, source as PlayerData),
            "player" => GetPlayerAttributeValue(pathParts, context),
            _ => null
        };
    }
    // 辅助方法：读取玩家自身属性
    private object GetSelfAttributeValue(string[] pathParts, PlayerData self)
    {
        if (pathParts.Length < 3 || self == null) return null;
        var attrName = pathParts[2];
        return attrName switch
        {
            "HandCount" => self.Hand.Count,
            "AttackPower" => self.AttackPower,
            "Agility" => self.Agility,
            "CurrentHp" => self.CurrentHp,
            "CurrentCost" => self.CurrentCost,
            "MaxHp" => self.MaxHp,
            _ => null
        };
    }
    // 辅助方法：解析四则运算表达式
    private object ResolveExpression(string expression, BattleContext context, Entity source, List<EffectResult> preEffectResults)
    {
        // 1. 替换表达式中的路径变量为实际数值
        // 2. 安全执行加减乘除四则运算，禁止复杂逻辑，防止注入
        // 3. 返回计算结果
        return 0;
    }
    // 其余辅助方法略
}
```

### 4.11 核心行为前置拦截机制（完善版）
#### 核心定位
统一管理所有玩家核心操作的权限校验，实现配置化的行为限制，彻底解决硬编码黑盒问题。
#### 标准化执行流程
所有核心行为执行前，必须先执行以下校验流程，只有校验通过才能继续执行：
```csharp
/// <summary>
/// 战斗结算引擎核心方法：核心行为前置校验
/// </summary>
public class BattleSettleEngine
{
    public bool PreActionCheck(BattleContext context, BattleCoreAction action, Entity source, List<Entity> targets = null)
    {
        // 0. 卡牌打出前，优先校验卡牌自身的UseLimit条件组
        if (action == BattleCoreAction.PlayCard || action == BattleCoreAction.PlayInstantCard || action == BattleCoreAction.PlayStrategyCard)
        {
            var card = source as CardData;
            if (card?.UseLimit != null && !context.ConditionChecker.Check(context, card.UseLimit, source))
            {
                // 记录拦截日志
                context.BattleLogs.Add(new BattleLogItem
                {
                    BattleId = context.BattleId,
                    Round = context.CurrentRound,
                    Phase = context.CurrentPhase,
                    EventType = BattleEventType.ActionBeforeExecute,
                    SourceId = card.CardId,
                    EventArgs = new Dictionary<string, object> { { "BlockAction", action }, { "BlockReason", "CardUseLimitNotMatch" } }
                });
                return false;
            }
        }
        // 1. 构造拦截事件
        var eventData = new BattleEvent
        {
            EventType = BattleEventType.ActionBeforeExecute,
            TriggerRound = context.CurrentRound,
            TriggerPhase = context.CurrentPhase,
            SourceId = source.Id,
            TargetIds = targets?.Select(t => t.Id).ToList(),
            ActionType = action
        };
        // 2. 先写入日志，再派发事件
        context.EventBus.Dispatch(eventData);
        // 3. 获取所有绑定该行为的拦截触发器，按优先级排序
        var blockTriggers = context.TriggerManager.GetBlockTriggers(action, source);
        // 4. 逐个执行触发器条件校验，任意一个触发器满足拦截条件则终止行为
        foreach (var trigger in blockTriggers)
        {
            if (context.ConditionChecker.Check(context, trigger.TriggerCondition, source))
            {
                // 记录拦截日志，返回false拦截行为
                context.BattleLogs.Add(new BattleLogItem
                {
                    BattleId = context.BattleId,
                    Round = context.CurrentRound,
                    Phase = context.CurrentPhase,
                    EventType = BattleEventType.ActionBeforeExecute,
                    SourceId = trigger.TriggerId,
                    EventArgs = new Dictionary<string, object> { { "BlockAction", action }, { "BlockReason", "TriggerConditionMatch" } }
                });
                return false;
            }
        }
        // 5. 无拦截，允许执行行为
        return true;
    }
}
```
#### 核心规则
1.  所有核心行为必须先执行`PreActionCheck`，校验不通过禁止执行
2.  拦截触发器优先级高于普通触发器，确保拦截逻辑先执行
3.  行为拦截仅终止当前操作，不回滚已执行的前置逻辑（如费用扣除）
4.  所有行为限制效果必须通过该机制实现，禁止在业务代码中硬编码判断

## 五、标准结算时序流程（强制锁死，补全版）
### 5.1 标准回合完整流程
```
1. 回合切换阶段
   ├─ 更新当前回合归属玩家ID
   ├─ 派发【RoundStart】事件
   ├─ 触发回合开始触发器/Buff（含中毒/灼烧DOT伤害结算，按优先级执行）
   ├─ 更新当前回合数
   ├─ 重置回合维度统计值
   └─ 清空玩家本回合卡牌打出记录，重置回响型态第一张牌标记
2. 抽牌阶段
   ├─ 执行【DrawCard】核心行为前置校验
   ├─ 校验通过则执行抽牌逻辑
   └─ 派发【CardDraw】事件
3. 费用刷新阶段
   ├─ 刷新玩家费用
   └─ 派发【CostChange】事件
4. 主操作阶段
   ├─ 玩家可打出瞬策/定策牌
   ├─ 打出卡牌前执行【PlayCard】核心行为前置校验（优先校验卡牌UseLimit）
   ├─ 校验手动选择目标合法性
   ├─ 瞬策牌打出后立即进入结算流程
   ├─ 定策牌打出后进入定策区锁定
   └─ 卡牌打出后，触发回响型态等卡牌复刻效果
5. 定策结算阶段
   ├─ 锁定玩家操作，禁止打出卡牌
   ├─ 按「层级>优先级>打出顺序」全局排序所有定策效果原子
   ├─ 反制层结算 → 标记失效卡牌/效果
   ├─ 防御层结算
   ├─ 伤害层结算
   ├─ 资源修改层结算
   ├─ Buff管理层结算
   └─ 特殊规则层结算
6. 回合结束阶段
   ├─ 派发【RoundEnd】事件
   ├─ 触发回合结束触发器/Buff
   ├─ 执行状态牌回合结束触发效果（如灼伤自伤）
   ├─ 按过期规则清理状态牌
   ├─ 清理过期触发器/Buff
   ├─ 清空回合临时数据
   └─ 重置回合统计值
7. 胜负判定阶段
   ├─ 检查玩家生命状态
   └─ 满足结束条件则派发【BattleEnd】事件
```
### 5.2 瞬策牌结算流程
```
1. 打出前校验：检查使用限制、费用、目标合法性，不通过则终止
2. 执行【PlayInstantCard】核心行为前置校验，校验不通过则终止
3. 扣除费用，派发【CardPlay】事件，记录本回合卡牌打出记录
4. 锁定卡牌效果，开放反制窗口，派发【EffectBeforeExecute】事件
5. 被反制 → 按配置进入坟场/消耗区，终止结算
6. 未被反制 → 按优先级排序效果原子，逐个执行
   ├─ 动态参数解析 → 数值修正 → 条件校验 → 目标解析
   ├─ 执行前事件派发 → 合法性校验 → 快照留存
   ├─ 循环执行核心逻辑 → 结果记录
   └─ 执行后事件派发
7. 所有效果执行完成，派发【CardSettleComplete】事件
8. 按配置进入坟场/消耗区
```
### 5.3 定策牌结算流程
```
1. 打出前校验：检查使用限制、费用、目标合法性，不通过则终止
2. 执行【PlayStrategyCard】核心行为前置校验，校验不通过则终止
3. 扣除费用，派发【CardPlay】事件，记录本回合卡牌打出记录
4. 卡牌进入定策区，锁定效果原子、层级、优先级，注册到定策结算队列
5. 等待进入定策结算阶段
6. 结算阶段按全局排序规则，逐层逐个执行效果原子
   └─ 执行流程同瞬策牌效果执行步骤
7. 单卡牌所有效果执行完成，派发【CardSettleComplete】事件
8. 按配置进入坟场/消耗区
9. 所有定策牌结算完成，退出定策结算阶段，解锁玩家操作
```

## 六、异常处理与边界防护规则（补全版）
1.  **空值防护**：目标解析无合法目标时，按`CanBeInvalid`配置执行，避免空指针崩溃
2.  **无限循环防护**：同一事件链触发器最大触发次数上限100次，超出自动中断，记录异常
3.  **循环执行防护**：效果原子循环执行最大次数上限100次，超出自动终止，记录异常
4.  **卡牌复刻循环防护**：单卡牌单回合最大复刻次数上限10次，超出自动终止，记录异常
5.  **数值溢出防护**：所有战斗数值设置上下限（如生命最低0、最高9999），避免数值溢出
6.  **脏数据防护**：战斗上下文仅结算引擎、Handler可写，其余模块只读，禁止直接修改数据
7.  **时序锁防护**：定策结算阶段锁定玩家操作，禁止打出任何卡牌，避免结算时序被打乱
8.  **动态参数异常防护**：动态参数解析失败时，使用默认静态参数，不终止效果执行，记录警告日志
9.  **表达式安全防护**：动态参数解析器仅支持加减乘除四则运算，禁止循环、条件判断、函数调用等复杂逻辑，防止代码注入
10. **手动选牌双重校验**：卡牌打出前、结算前，双重校验手动选择的目标是否仍在指定区域、是否合法，不合法则终止结算
11. **永久触发器上限防护**：单场战斗的全局永久触发器设置数量上限（默认50个），超出上限禁止注册，防止性能问题
12. **状态牌数量上限防护**：单玩家单区域状态牌数量上限20张，超出上限自动销毁最早生成的状态牌，防止性能问题
13. **异常回滚机制**：单效果执行失败可回滚，全局致命异常回滚到上一回合结束的完整快照
14. **调试模式**：支持战斗步骤回放、断点调试、日志可视化，方便排查配置与逻辑问题
