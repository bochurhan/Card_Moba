## 二、Sprint 2.1: 配置系统（建议第一个完成）

### 目标
将卡牌数据从代码中抽离，通过 JSON 配置表管理。

### 为什么优先做这个？
- 后续添加新卡牌不需要改代码
- 可以快速测试不同数值平衡
- 为策划填表打基础

### 具体任务

#### 2.1.1 创建卡牌 JSON 配置文件
```
Config/Json/Cards/cards.json
```

示例格式：
```json
{
  "cards": [
    {
      "cardId": 1001,
      "cardName": "火球术",
      "description": "造成{value}点伤害",
      "trackType": "Instant",
      "tags": "Damage",
      "targetType": "CurrentEnemy",
      "energyCost": 1,
      "rarity": 1,
      "effectsRef": "E1001-1"
    }
  ],
  "effects": [
    {
      "effectId": "E1001-1",
      "effectType": 401,
      "value": 4,
      "description": "造成4点伤害"
    }
  ]
}
```

> **注意**：`effectsRef` 使用 `E` 前缀以避免 Excel 将其误解析为日期格式。

#### 2.1.2 实现 CardConfigLoader
位置：`Client/Assets/Scripts/Data/ConfigData/CardConfigLoader.cs`

功能：
- 从 `StreamingAssets/Config/Cards/cards.json` 读取
- 解析 JSON 为 `List<CardConfig>`
- 提供 `GetCardById(int id)` 查询接口

#### 2.1.3 修改 BattleGameManager
- 删除 `CreateTestDeck()` 中的硬编码
- 改为从 `CardConfigLoader` 获取卡牌

### 预计工作量
2-3 小时

## 四、Sprint 2.3: AI 增强

### 目标
让 AI 更有挑战性，不只是"用完能量"。

### 具体任务

#### 2.3.1 AI 策略系统重构

创建 `Client/Assets/Scripts/GameLogic/AI/SimpleAI.cs`：

```csharp
public class SimpleAI
{
    // 评估每张手牌的价值
    public int EvaluateCard(CardConfig card, PlayerBattleState self, PlayerBattleState enemy);
    
    // 决定这回合要打哪些牌
    public List<int> DecideCardsToPlay(PlayerBattleState self, PlayerBattleState enemy);
}
```

#### 2.3.2 AI 决策逻辑

**优先级规则**：
1. 如果对手 HP ≤ 自己手牌总伤害 → 全力输出斩杀
2. 如果自己 HP ≤ 10 → 优先防御/回复
3. 否则 → 平衡出牌

**卡牌价值评估**：
- 伤害牌：伤害值 / 费用
- 防御牌：(自己缺失HP比例) × 护盾值 / 费用
- 回复牌：(自己缺失HP比例) × 回复值 / 费用

#### 2.3.3 AI 难度等级

```csharp
public enum AIDifficulty
{
    Easy,    // 随机出牌
    Normal,  // 基础策略
    Hard,    // 斩杀线判断 + 资源规划
}
```

## 六、开发优先级建议

对于新手开发者，我建议按以下顺序进行：

### 第一阶段：打好基础（1-2天）
1. **阅读架构文档** — 理解分层和模块职责
2. **运行游戏并调试** — 在代码中加断点，观察数据流
3. **做一个小修改** — 比如修改卡牌伤害值，确认理解了流程

### 第二阶段：配置系统（半天）
4. **Sprint 2.1** — 实现 JSON 配置加载

### 第三阶段：内容扩展（1天）
5. **Sprint 2.2** — 添加新卡牌类型
6. **设计并测试卡组平衡**

### 第四阶段：体验优化（1天）
7. **Sprint 2.3** — AI 策略增强
8. **Sprint 2.4** — UI 动画和反馈

### 总计预估时间
**4-5 天**（每天 2-3 小时）

## 九、遇到问题怎么办？

1. **编译错误** → 查看错误信息，通常是拼写或类型问题
2. **运行时报错** → 查看 Unity Console，定位到具体代码行
3. **逻辑不对** → 添加 `Debug.Log()` 打印变量值
4. **不知道怎么实现** → 先写伪代码，再一步步实现

### Debug 小技巧
```csharp
// 打印当前函数被谁调用
Debug.Log($"[模块名] 方法名被调用，参数={变量}");

// 条件断点（在 Visual Studio / Rider 中设置）
// 只在特定条件下暂停
```
