# 局内构筑系统详解

**文档版本**：V1.0  
**前置阅读**：[Overview.md](Overview.md)  
**阅读时间**：6 分钟  
**最后更新**：2026年03月22日

## 🎯 设计目标

局内构筑系统需要同时满足以下要求：

1. **强化每局差异性**：让每局对局在卡组层面产生可感知的成长轨迹；
2. **服务整局节奏**：构筑窗口与阶段结构自然衔接，不打断战斗节奏；
3. **不压过局外预构筑**：局内随机性不应彻底覆盖局外身份与风格；
4. **不引入过高操作负担**：选项数量克制，决策清晰；
5. **能与奖励系统自然衔接**：团队目标奖励可影响候选池倾向。

---

## 🪟 构筑窗口机制

首版局内构筑采用"**阶段结算后统一调整**"的方式，而不是在战斗中进行修改。

- 每场战斗 / 阶段结束后，进入一个短暂的构筑窗口；
- 玩家可在该窗口中执行一次局内构筑操作；
- 各操作选项默认**互斥**，只能选择其中一种；
- 若该阶段战斗中成功产生击杀，则可**额外多选 1 个选项**（击杀加选，见下文）。

### 窗口数量

基于当前整局 5 阶段结构，首版设置 **4 次局内构筑窗口**：

| 时机 | 说明 |
|------|------|
| 第 1 场结束后 | 第一次构筑调整 |
| 第 2 场结束后 | 第二次构筑调整 |
| 第 3 场结束后 | 第三次构筑调整 |
| 第 4 场结束后 | 死斗前最后一次调整 |

> 死斗阶段（第 5 场）前不再提供构筑窗口。

## 🃏 四类操作

每个构筑窗口采用 **4 选 1** 的形式，四类操作固定如下：

### 操作 A：拿卡

- 从策划配置的候选池中选择 **1 张牌**加入卡组；
- 拿卡默认采用 **3 选 1**；
- 候选池按英雄、阶段、团队目标奖励等条件分层配置；
- 用于强化当前路线、补足短板或向某个 build 倾斜。

**稀有度分布（白:蓝:金 = 6:3:1）**：
- 白色牌（60%）：稳定普通补强，不改变 build 方向；
- 蓝色牌（30%）：有一定塑形能力的中级选项；
- 金色牌（10%）：可能明显改变本局路线的高价值选项。

### 操作 B：删卡

- 从当前卡组中移除 **1 张牌**；
- 主要用于去除低效基础牌或与当前路线不匹配的组件；
- 每次删卡仅可删除 1 张；
- 大部分牌都允许被删除；
- 对小卡组体系尤其重要，能显著提升 build 纯度。

**删卡收益曲线**：
- 牌组较"厚"时，删 1 张的收益相对温和；
- 当牌组逐渐成型、关键牌密度提升后，继续删卡会带来更明显的提纯收益；
- 删卡被视为"前期较稳、后期可能突增"的 build 操作。

### 操作 C：升级

- 选择当前卡组中的 **1 张牌**进行升级；
- 每次升级仅能升级 1 张；
- 绝大部分牌都可以升级；
- 升级形式由策划预设。

**升级类型分类**：
- **数值强化型**：如提高伤害/护盾数值、降低费用等；
- **机制强化型**：如新增次级效果、改变结算桶位、强化标签联动等；
- 不同卡牌的升级效果并不统一，用于突出 build 路径差异与核心牌成型感。

### 操作 D：回血

- 恢复角色生命值 = **当前最大生命值的 30%**；
- 仅作用于角色生命值，首版不恢复核心状态；
- 意图让回血成为明确可选项，而非纯保底弱选项；
- 也是控制"速杀流 vs 苟命流"风险收益博弈的重要杠杆。

---

## ⚔️ 击杀加选规则

- 若团队在本阶段战斗中产生至少 1 次击杀，则在本次构筑窗口中可**额外多选 1 项操作**；
- 加选判定基于**团队本阶段表现**，而非仅限当前分路；
- 额外操作的类型不限，可与基础操作重复选择同类（首版暂定，待测试确认是否需要限制）。

---

## 📦 候选池配置原则

拿卡与升级都依赖候选池设计，首版采用**策划配置池子**而非完全随机大池：

- 每个英雄有专属的职业牌候选池；
- 另有部分通用牌候选池（所有英雄可见）；
- 不同阶段可开放不同稀有度或不同功能倾向的候选；
- 团队目标奖励或阶段表现，也可影响候选池倾向（如某类奖励后开放特殊候选）。

---

## 📈 如何扩大局内 build 影响力

在"全局仅 4 次构筑窗口"的前提下，若希望进一步放大局内 build 的影响力，优先考虑以下方向：

| 方向 | 思路 |
|------|------|
| **A：提高单次选择的塑形强度** | 让升级更有个性；让删牌对小牌组流派有明显回报；让高稀有度拿牌真正改变本局路线 |
| **B：扩大构筑操作的连锁效应** | 某些牌或奖励对"拿卡/删卡/升级"产生协同，如"首次删牌获得额外小收益"或"升级后连带强化相关标签牌" |
| **C：通过候选池质量放大差异** | 不增加窗口次数，而是让候选池按阶段/职业/目标/前期表现发生倾向性变化，使每次 3 选 1 的含义更重 |
| **D：增加轻量额外构筑资源** | 阶段胜利、击杀、团队目标达成后给予额外升级点/删牌券/拿牌优先权等，在现有窗口内消耗 |

## ⚠️ 首版待细化问题

以下问题会直接影响后续数值配置与测试：

- 初始牌组数量是否需要进一步调整，以适应仅有 4 次构筑窗口的节奏；
- 白/蓝/金三档稀有度分别在候选池中承担怎样的功能定位；
- 删卡收益突增的拐点大致出现在牌组规模的哪个区间；
- 哪些牌允许机制升级，哪些牌只允许数值升级；
- 回血 30% 是否会在死斗前成为过强的默认选项；
- 击杀加选是否需要限制"额外操作不能与基础操作重复同类"；
- 团队目标奖励与构筑窗口之间是否会形成过强联动；
- 是否需要额外的轻量构筑资源来放大局内 build 影响力。
